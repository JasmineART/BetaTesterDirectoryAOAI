Beta Test Instructions

Thank you for testing the **OA AI Tracker**. Your feedback is crucial for validating the user experience, authentication, and the core data processing workflow.

1. Access and Login

  • **Testing Environments:** Please test on a modern **desktop** browser AND a **mobile device** (iOS/Android). Note any differences in layout or functionality.
  • **Authentication:** Locate and use the **Google Login** feature to sign in.
  • **Accessible Login Check:** Verify the usability of the login features, especially regarding assistive technologies (screen readers, keyboard navigation). Report any accessibility issues immediately.

2. Workflow & Data Import

  1. **Create a New Project:** After signing in, navigate to the **Projects** section/tab. Locate and click the button to **Add a New AI Project** (or similar naming). Fill in any required project details.
  2. **Data Connection and Upload:** Within the new project, navigate to the data upload area. Select the provided Google Sheets file (or a file of a different type) to import the data.
    • **Check File Types:** Specifically verify that importing the native **Google Sheets file type** works correctly.
    • **Explore Connections:** If available, check other data connection features as desired (e.g., live API, database, CSV, JSON).

3. Testing Objectives

Area Testing Focus
Dashboard Review **Go to the Dashboards section.** Review that the populated data makes sense. Ensure all charts, tables, and readiness scores are calculated and displayed correctly.
Functionality Test all filters, sorting options, and interactive elements. Data must update instantly and accurately across all visualizations.
Usability & UX Assess design/layout on **Desktop and Mobile**. Are charts legible? Are navigation links clear and working?
Performance Note the speed of login, project creation, data import, and dashboard loading.

4. Feedback & Bug Reporting

**All ideas and feedback are welcome!** Please document all issues with the following structure. You can report feedback via the provided Discord link.

  • **Issue Type:** (Bug, Enhancement Suggestion, Usability Feedback)
  • **Severity:** (Critical, High, Medium, Low)
  • **Description of the Issue:** A clear explanation of what happened.
  • **Steps to Reproduce:** (1) Go to X page. (2) Click Y button. (3) Observe Z error/result.
  • **Browser & OS/Device:** (e.g., Chrome on Windows 10, Safari on iPhone 14)
